# wb-order-reporter

## Как запустить

```bash
pip install -r requirements.txt
cp .env.example .env
python main.py
```

**Переменные окружения:**

| Переменная | Описание |
|---|---|
| `WB_API_TOKEN` | Токен статистики Wildberries |
| `TELEGRAM_BOT_TOKEN` | Токен бота от @BotFather |
| `TELEGRAM_CHAT_ID` | ID чата |

## Хранилище

Для тестового задания выбрала SQLite — не требует сервера, разворачивается мгновенно, держит 100к+ строк без проблем. При работе с AI через SQLite нужен ручной шаг: выгрузить нужные строки SQL запросом, экспортировать в текст или CSV и передать в контекст — это работает, но неудобно в автоматизации. Для продакшна с AI интеграцией оптимальный выбор в 2026 году — **Supabase**. Claude и ChatGPT поддерживают Supabase через MCP (Model Context Protocol): AI инструмент подключается напрямую к базе и делает SQL запросы без ручного экспорта. Supabase также имеет встроенный pgvector — можно хранить эмбеддинги рядом с данными заказов и делать семантический поиск. Альтернативы: PostgreSQL (нужен отдельный сервер), ClickHouse (хорош для аналитики, но избыточен здесь), CSV-only (нет индексов и запросов).

## Что доделала бы для продакшна

- **Планировщик с контролем пропусков**: cron или APScheduler + запись последнего успешного запуска в БД — если скрипт не отработал, это видно сразу, а не через неделю когда данных нет
- **Бэкфилл**: сейчас можно передать любую дату через `target_date`, но нет CLI аргумента — добавила бы `--date 2026-06-15` для ручного перезапуска за конкретный день без правки кода
- **Валидация схемы API**: оборачивать сырой ответ WB в Pydantic модель, чтобы при изменении API падать сразу с понятной ошибкой, а не тихо терять поля
- **Поддержка нескольких магазинов**: сейчас один токен = один магазин; для мультиаккаунта нужен цикл по токенам с изоляцией данных по `store_id` в БД
- **Миграция на PostgreSQL/Supabase**: SQLite не держит конкурентные записи; Supabase даёт REST API поверх базы — AI инструменты дёргают данные напрямую без промежуточного экспорта
- **Разделение пайплайна и уведомлений**: сейчас Telegram блокирует завершение скрипта — в продакшне данные и нотификации должны быть независимы, уведомления в отдельную очередь с ретраями
